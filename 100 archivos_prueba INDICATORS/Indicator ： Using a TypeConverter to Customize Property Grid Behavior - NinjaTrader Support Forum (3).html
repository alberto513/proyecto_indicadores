    Indicador: Usando um TypeConverter para personalizar o comportamento da grade de propriedades

    23/03/2017, 10:05 AM
    O comportamento padro da grade de propriedades do NinjaTrader 8 projetado para lidar com a exibio mais bsica de suas propriedades personalizadas de Indicador e Estratgia. No entanto, h casos especiais em que voc pode querer controlar como as propriedades exclusivas so exibidas para outros usurios. Como usar um TypeConverter mais um conceito geral do C# usado para converter valores entre tipos de dados (uma string para um enum, por exemplo), as possibilidades do que voc pode fazer esto dentro dos limites do .NET Framework. Este exemplo do NinjaScript foi produzido para ajudar os desenvolvedores do NinjaScript a entender como aproveitar as classes auxiliares IndicatorBaseConverter e StrategyBaseConverter para personalizar o comportamento da grade de propriedades sem afetar o comportamento geral das propriedades do NinjaTrader 8.

    Observao:
    • O NinjaTrader 8 espera que as propriedades personalizadas sejam do tipo valor ou tipo que implemente a interface ICloneable
    • Este exemplo de referncia pressupe que voc esteja familiarizado com as prticas bsicas do C# TypeConverter e tem como objetivo ser um ponto de partida. Existem guias abrangentes de terceiros disponveis on-line que podem ajudar voc a implementar uma meta especfica de conversor de tipo no coberta neste exemplo


    Conceitos-chave neste exemplo:

    H 5 casos de uso demonstrados neste exemplo que se enquadram em duas categorias diferentes:

    1. Manipular dinamicamente o que exibido na Grade de Propriedades da IU
    1. Mostrar/ocultar propriedades com base na entrada secundria
    2. Desabilitar/habilitar propriedades com base na entrada secundria


    2. Personalizando como uma propriedade exibida
    1. Exibir valores de enumerao "Amigveis"
    2. Reimplementar um CheckBox "bool" como um ComboBox "amigvel"
    3. Exibir uma coleo/lista personalizada com valores definidos pelo usurio em tempo de execuo
