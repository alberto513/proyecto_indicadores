    Indicador: Usando um TypeConverter para personalizar o comportamento da grade de propriedades

    23/03/2017, 10:05 AM
    O comportamento padrão da grade de propriedades do NinjaTrader 8 é projetado para lidar com a exibição mais básica de suas propriedades personalizadas de Indicador e Estratégia. No entanto, há casos especiais em que você pode querer controlar como as propriedades exclusivas são exibidas para outros usuários. Como usar um TypeConverter é mais um conceito geral do C# usado para converter valores entre tipos de dados (uma string para um enum, por exemplo), as possibilidades do que você pode fazer estão dentro dos limites do .NET Framework. Este exemplo do NinjaScript foi produzido para ajudar os desenvolvedores do NinjaScript a entender como aproveitar as classes auxiliares IndicatorBaseConverter e StrategyBaseConverter para personalizar o comportamento da grade de propriedades sem afetar o comportamento geral das propriedades do NinjaTrader 8.

    Observação:
    • O NinjaTrader 8 espera que as propriedades personalizadas sejam do tipo valor ou tipo que implemente a interface ICloneable
    • Este exemplo de referência pressupõe que você esteja familiarizado com as práticas básicas do C# TypeConverter e tem como objetivo ser um ponto de partida. Existem guias abrangentes de terceiros disponíveis on-line que podem ajudar você a implementar uma meta específica de conversor de tipo não coberta neste exemplo


    Conceitos-chave neste exemplo:

    Há 5 casos de uso demonstrados neste exemplo que se enquadram em duas categorias diferentes:

    1. Manipular dinamicamente o que é exibido na Grade de Propriedades da IU
    1. Mostrar/ocultar propriedades com base na entrada secundária
    2. Desabilitar/habilitar propriedades com base na entrada secundária


    2. Personalizando como uma propriedade é exibida
    1. Exibir valores de enumeração "Amigáveis"
    2. Reimplementar um CheckBox "bool" como um ComboBox "amigável"
    3. Exibir uma coleção/lista personalizada com valores definidos pelo usuário em tempo de execução
